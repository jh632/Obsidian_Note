# ESP32-C3 音乐播放器课程设计文档

  

## 1. 设计目标

  



  

本方案重点满足以下模块要求：

  

- 显示模块电路验证

- DAC 模块电路验证

- 功放模块电路验证

- 音频模块电路验证

- 人机交互模块电路验证

  

同时考虑到制作者以软件实现为主、硬件焊接经验有限，方案优先采用 `ESP32-C3-WROOM-02` 模组作为主控，降低最小系统设计和焊接风险。

  

## 2. 方案选择说明

  

### 2.1 主控芯片选择

  

本设计选择 `ESP32-C3-WROOM-02` 作为主控模块，原因如下：

  

- 采用官方模组，内部已集成晶振和 Flash，外围电路更简单

- 支持 `SPI`、`I2C`、`I2S`、`UART` 等接口，便于连接显示、TF 卡和音频 DAC

- 相比裸 `STM32F103C8T6`，模组方式更适合首次自画板实现

- 软件生态完善，后续程序编写和调试效率更高

  

### 2.2 方案取舍

  

本设计不采用 `DFPlayer Mini` 作为核心音频方案，原因是该模块将音频存储、解码、DAC、功放功能集成在一起，不利于按照课程要求拆分成独立模块进行分析和验证。

  

由于 `ESP32-C3` 不带片上 DAC，因此本设计增加独立 `PCM5102A` DAC 模块，用于完成 DAC 模块验证。

  

## 3. 系统总体方案

  

系统总体结构如下：

  

`TF 卡模块 -> ESP32-C3 主控 -> PCM5102A DAC 模块 -> PAM8403 功放模块 -> 扬声器`

  

同时：

  

- `ESP32-C3 -> OLED 显示模块`

- `按键模块 -> ESP32-C3`

  

### 3.1 系统功能

  

系统实现以下基本功能：

  

- 从 TF 卡读取音频文件

- 播放 WAV 音频

- OLED 显示当前播放状态

- 通过按键实现播放/暂停、上一曲、下一曲

- 音频经 DAC 转换后送入功放，最终驱动扬声器发声

  

## 4. 模块划分

  

### 4.1 显示模块

  

显示模块选用 `0.96 寸 OLED`，驱动芯片采用 `SSD1306`，接口采用 `I2C`。

  

选择理由：

  

- 接线简单，仅需 `SDA` 与 `SCL` 两根信号线

- 占用 IO 少

- 能直观显示曲目编号、播放状态、音量等信息

  

显示内容建议包括：

  

- 系统启动界面

- 当前歌曲编号

- 播放/暂停状态

- 音量等级

  

### 4.2 音频模块

  

音频模块采用 `TF 卡模块` 或板载 `MicroSD 卡座`。

  

主要作用：

  

- 存储音频文件

- 通过 `SPI` 接口向主控提供音频数据

  

选择理由：

  

- 硬件简单，成本低

- 文件更换方便

- 便于说明“音频模块”与“DAC 模块”的功能边界

  

建议音频格式：

  

- 优先使用 `WAV`

  

原因：

  

- 软件解码难度低

- 更适合课程设计快速实现

  

### 4.3 DAC 模块

  

DAC 模块采用 `PCM5102A`。

  

主要作用：

  

- 接收来自 `ESP32-C3` 的 `I2S` 数字音频数据

- 转换为模拟音频信号输出给功放模块

  

选择理由：

  

- 与 `ESP32-C3` 的 `I2S` 外设匹配

- 模块独立，便于单独分析和答辩

- 相比自己设计分立 DAC 电路更稳妥

  

### 4.4 功放模块

  

功放模块采用 `PAM8403`。

  

主要作用：

  

- 对 DAC 输出的模拟音频进行功率放大

- 驱动小功率扬声器发声

  

选择理由：

  

- 常见、便宜、资料多

- 5V 供电，应用简单

- 能较好地完成课程中的功放模块验证

  

### 4.5 人机交互模块

  

人机交互模块采用 `3 个独立按键`。

  

按键功能建议如下：

  

- `KEY1`：上一曲

- `KEY2`：播放/暂停

- `KEY3`：下一曲

  

如需增强交互功能，可增加：

  

- 音量加键

- 音量减键

- 模式切换键

  

但从效率角度出发，首版建议仅保留 3 个按键。

  

## 5. 硬件总体设计

  

### 5.1 电源设计

  

建议系统输入电源为 `5V`，再通过 `3.3V LDO` 给 `ESP32-C3` 和低压逻辑模块供电。

  

供电关系如下：

  

- `5V`：PAM8403、部分外围模块

- `3.3V`：ESP32-C3、OLED、TF 卡、PCM5102A 逻辑侧

  

设计要点：

  

- 电源输入端增加滤波电容

- 3.3V 输出端增加去耦电容

- 每个关键芯片电源引脚附近放置 `0.1uF` 去耦电容

  

### 5.2 ESP32-C3 最小系统

  

自画板时，`ESP32-C3-WROOM-02` 最小系统建议包括：

  

- `3.3V` 电源输入

- `EN` 上拉电阻

- 启动下载相关引脚处理

- 去耦电容

- 串口下载接口

  

设计时应重点注意：

  

- 保证模组天线区域下方和前方尽量净空

- 避免在天线区域铺铜和走高速线

- 下载相关引脚避免被外部电路强拉高或强拉低

  

### 5.3 接口分配建议

  

以下为一套便于实现的参考引脚分配，实际画图前应结合 `ESP32-C3` 启动脚要求再次检查：

  

#### OLED 显示模块

  

- `GPIO4` -> `SDA`

- `GPIO5` -> `SCL`

  

#### PCM5102A DAC 模块

  

- `GPIO6` -> `BCK`

- `GPIO7` -> `LRCK/WS`

- `GPIO8` -> `DIN`

  

#### TF 卡模块

  

- `GPIO2` -> `SCK`

- `GPIO3` -> `MOSI`

- `GPIO10` -> `MISO`

- `GPIO1` -> `CS`

  

#### 按键模块

  

- `GPIO18` -> `KEY1`

- `GPIO19` -> `KEY2`

- `GPIO20` -> `KEY3`

  

## 6. 原理图设计说明

  

### 6.1 显示模块原理图说明

  

OLED 模块通过 `I2C` 与主控连接，需要连接：

  

- `VCC`

- `GND`

- `SDA`

- `SCL`

  

若模块板未自带上拉，可增加 `4.7k` 上拉电阻到 `3.3V`。

  

### 6.2 音频模块原理图说明

  

TF 卡模块采用 `SPI` 连接方式，需要连接：

  

- `CS`

- `SCK`

- `MOSI`

- `MISO`

- `VCC`

- `GND`

  

若采用板载卡座，应注意电平兼容和电源滤波。

  

### 6.3 DAC 模块原理图说明

  

`PCM5102A` 主要连接信号包括：

  

- `BCK`

- `LRCK`

- `DIN`

- `VCC`

- `GND`

  

输出侧为模拟音频信号：

  

- `L_OUT`

- `R_OUT`

  

本设计可采用单声道或双声道方案。若仅驱动单个扬声器，可只使用单声道输入功放，或左右声道先做简单混合后送入功放。

  

### 6.4 功放模块原理图说明

  

`PAM8403` 输入端接收 DAC 输出模拟音频，输出端连接扬声器。

  

需要连接：

  

- `VIN/5V`

- `GND`

- `L_IN/R_IN`

- `SPK+`

- `SPK-`

  

若使用现成功放小模块，则建议在自画 PCB 上预留标准排针接口，减少焊接难度。

  

### 6.5 按键模块原理图说明

  

按键采用最简单接法：

  

- 一端接 `GPIO`

- 一端接 `GND`

  

软件中开启内部上拉，按下时读取为低电平。

  

如需增强稳定性，可增加硬件消抖网络，但首版可优先使用软件消抖。

  

## 7. PCB 设计建议

  

### 7.1 布局建议

  

建议按功能模块分区布局：

  

- 左上：电源模块

- 中间：ESP32-C3 主控模块

- 右上：OLED、按键接口

- 左下：TF 卡模块

- 中下：PCM5102A DAC 模块

- 右下：PAM8403 功放与扬声器接口

  

### 7.2 走线建议

  

- 数字电路与模拟音频路径尽量分开

- `I2S` 走线尽量短且清晰

- 功放输出到扬声器的走线适当加宽

- 模拟音频输入线尽量远离 Wi-Fi 天线区和高速数字线

  

### 7.3 天线区域注意事项

  

由于 `ESP32-C3-WROOM-02` 使用板载天线，PCB 设计时要注意：

  

- 天线前方禁止放置金属器件

- 天线下方尽量不要铺铜

- 天线区域周边避免密集走线

  

## 8. 软件实现思路

  

软件可按以下流程工作：

  

1. 系统上电初始化

2. 初始化 OLED 显示

3. 初始化 TF 卡文件系统

4. 初始化 I2S 外设

5. 初始化按键扫描任务

6. 从 TF 卡读取 WAV 文件

7. 通过 I2S 将音频数据发送到 `PCM5102A`

8. 更新 OLED 显示状态

9. 响应按键切歌、暂停等操作

  

## 9. 主要器件清单建议

  

### 9.1 核心器件

  

- `ESP32-C3-WROOM-02`

- `PCM5102A`

- `PAM8403`

- `0.96 寸 OLED`

- `TF 卡座` 或 `TF 卡模块`

- `8Ω 3W 扬声器`

  

### 9.2 常用外围器件

  

- `3.3V LDO`

- `Type-C` 供电接口或 DC 供电端子

- `0.1uF` 去耦电容若干

- `10uF/22uF` 电容若干

- `10k` 电阻若干

- 独立按键 `3` 个

- 排针、端子若干

  

## 10. 方案优点与风险

  

### 10.1 方案优点

  

- 满足课程要求的模块划分

- 主控采用模组，降低硬件设计门槛

- 软件结构清晰，便于分阶段调试

- 可在答辩中清楚说明各模块功能边界

  

### 10.2 主要风险

  

- `ESP32-C3` 不带片上 DAC，必须依赖外部 DAC 模块

- 若自行焊接 `PCM5102A`、`PAM8403` 裸芯片，焊接难度会上升

- 若直接播放 MP3，软件解码复杂度较高

  

### 10.3 风险控制建议

  

- 首版优先使用 `WAV` 音频格式

- 自画底板上尽量采用模组/模块插接方式

- 先独立验证 `OLED`、`TF 卡`、`I2S DAC`、`功放`，再做整合

  

## 11. 最终结论

  

综合课程要求、实现难度和制作效率，本设计推荐采用如下最终方案：

  

`ESP32-C3-WROOM-02 + OLED 显示模块 + TF 卡音频模块 + PCM5102A DAC 模块 + PAM8403 功放模块 + 按键交互模块`

  

该方案兼顾了：

  

- 模块边界清晰

- 原理图易于绘制

- 软硬件工作量可控

- 便于课程验收和报告撰写

  

若后续老师进一步要求“所有模块必须完全自画、不能接现成模块”，则可以在本方案基础上逐步将 `TF 卡模块`、`PCM5102A`、`PAM8403` 从接口式方案过渡为板载器件方案，但首版不建议直接采用高风险实现路径。
