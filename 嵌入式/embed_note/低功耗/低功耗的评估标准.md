# 低功耗的评估标准

## 概述

评估嵌入式系统的低功耗性能，需要从**电流指标、电池寿命、测量方法、能效模型**等多个维度进行量化分析。核心目标是：用数据驱动功耗优化决策，而非凭感觉判断"省不省电"。

## 核心指标

### 1. 电流指标

| 指标 | 含义 | 典型量级 | 说明 |
|------|------|----------|------|
| **Active Current** | 全速运行时电流 | mA ~ 100+ mA | CPU + 外设全部工作 |
| **Sleep Current** | 休眠模式电流 | nA ~ µA 级 | 越低越好，决定待机功耗 |
| **Peak Current** | 瞬时峰值电流 | 取决于负载 | 影响电池选型和电源设计 |
| **Average Current** | 周期平均电流 | µA ~ mA | 综合反映实际功耗水平 |

> **关键认知**：Sleep Current 决定了设备"睡着时能撑多久"，Active Current 决定了"干活时多费电"，而 **Average Current** 才是真正决定电池寿命的指标。

### 2. 电池寿命估算

```
电池寿命 (h) = 电池容量 (mAh) / 平均电流 (mA)
```

**平均电流的计算**：

```
平均电流 = Σ (各状态电流 × 该状态占空比)

例：设备每 60s 采集一次数据，Active 2s (20mA)，Sleep 58s (5µA)
平均电流 = 20mA × (2/60) + 0.005mA × (58/60)
         ≈ 0.667 + 0.0048
         ≈ 0.672 mA

使用 CR2032 纽扣电池 (220mAh)：
寿命 ≈ 220 / 0.672 ≈ 327 小时 ≈ 13.6 天
```

### 3. 能效指标

| 指标 | 公式 | 单位 | 适用场景 |
|------|------|------|----------|
| **能量/次操作** | 功率 × 操作时间 | µWh 或 mJ | 比较不同实现方案的效率 |
| **能耗比** | 能量 / 数据量 | J/bit 或 J/采样 | 通信模块效率评估 |
| **功耗密度** | 功率 / 功能 | mW/MIPS 或 mW/KB | 不同平台横向对比 |

## 测量方法

### 1. 仪器选择

| 工具 | 精度范围 | 优点 | 缺点 |
|------|----------|------|------|
| **高精度万用表** (如 Keysight 34461A) | pA ~ mA | 精度高，适合静态测量 | 采样率低，抓不住瞬态 |
| **电流探头 + 示波器** | mA 级 | 可抓瞬态波形 | 小电流精度不足 |
| **专用功耗分析仪** (Nordic PPK2, Monsoon) | nA ~ A | 自动量程切换，软件分析 | 价格较高 |
| **EnergyTrace (TI)** | µA 级 | 与 CCS 集成，适合 MSP430 | 仅限 TI 平台 |

### 2. 测量要点

- **串联采样电阻**：在 VCC 和系统之间串入小阻值电阻 (1Ω ~ 100Ω)，通过压降计算电流
- **注意旁路电容**：大电容会缓冲电流变化，导致测量值偏小 — 测量时需考虑电容充放电
- **采样率匹配**：测量 Sleep Current 可用低采样率；抓 Active 瞬态需要 ≥10kHz 采样率
- **温度影响**：Sleep Current 随温度升高而增大，datasheet 给出的是 25°C 典型值

### 3. 测量流程

```
1. 确定测量目标 → 想知道什么状态下的电流？
2. 选择合适工具 → 精度范围是否覆盖目标电流？
3. 断开目标模块 → 单独测量各模块功耗
4. 多次测量取平均 → 消除随机误差
5. 记录环境条件 → 温度、电压、固件版本
```

## ESP32 各模式电流参考

> 数据来源：ESP32 Datasheet，实际值因批次和温度有差异

| 模式 | 典型电流 | 说明 |
|------|----------|------|
| Active (240MHz, Wi-Fi ON) | ~80-180 mA | CPU 全速 + Wi-Fi 收发 |
| Modem Sleep (Wi-Fi ON) | ~15-20 mA | CPU 可调频，Wi-Fi 按需唤醒 |
| Light Sleep | ~0.8 mA | CPU 暂停，RAM 保持，可被 GPIO/定时器唤醒 |
| Deep Sleep | ~5-10 µA | 仅 RTC 控制器 + RTC Memory (8KB) 保持供电 |
| Deep Sleep + Flash DPD | <1 µA | SPI Flash 进入 Deep Power-Down 模式 |

## 功耗优化检查清单

- [ ] 是否启用了所有可用的 Sleep 模式？
- [ ] 未使用的 GPIO 是否配置为输入并关闭上下拉？
- [ ] 未使用的外设时钟是否已 gate？
- [ ] ADC/DAC 不使用时是否关闭？
- [ ] 电压调节器是否选择了低功耗型号？
- [ ] 电池选型是否匹配系统峰值电流需求？
- [ ] 软件是否避免了 busy-wait，改用中断/事件驱动？
- [ ] 通信模块是否在空闲时进入 Sleep/Power-down？
