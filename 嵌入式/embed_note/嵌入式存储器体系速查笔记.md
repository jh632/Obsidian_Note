# 嵌入式存储器体系速查笔记

## 1. 术语澄清
- **DDSRAM**：在工程语境中几乎特指 **DDR SDRAM** (双倍数据率同步动态随机存储器)。下文将其归于 DRAM 家族细述。

---

## 2. 存储器家族树 (按易失性与结构划分)

```text
存储器
├── 易失性 (掉电丢数据) → RAM
│   ├── SRAM (静态RAM)
│   │   └── 片上缓存、TCM、外部高速异步SRAM
│   ├── DRAM (动态RAM)
│   │   ├── SDRAM (同步DRAM)
│   │   └── DDR SDRAM (DDR1/2/3/4/5、LPDDR、GDDR)
│   │       └── 你问的“DDSRAM”就是这类
│   └── 伪静态RAM (PSRAM)：DRAM内核 + SRAM接口
│
└── 非易失性 (掉电保存) → ROM/Flash
    ├── 传统ROM
    │   ├── Mask ROM (出厂固化)
    │   ├── PROM (一次性编程)
    │   └── EPROM (紫外擦除，已基本淘汰)
    ├── EEPROM (电擦除，可按字节写)
    └── Flash (块擦除)
        ├── NOR Flash → 可XIP，存固件，MCU内部Flash即这类
        ├── NAND Flash → 大容量存数据，需要ECC
        │   ├── Raw NAND (裸片，需要Host做坏块管理)
        │   ├── eMMC (封装了控制器，标准接口)
        │   └── UFS (更高速的闪存存储)
        └── 串行/SPI Flash → 通常是NOR，用于外部固件存储
```

## 3. 各类存储器详解

### 3.1 SRAM (静态随机存取存储器)
- **结构**：六个晶体管构成一个锁存器。
- **特点**：只要上电，数据就稳定；无需刷新；读写速度极快，可匹配CPU主频；集成度低，成本高，功耗相对较大。
- **嵌入式位置**：
  - MCU 内部的通用 RAM（堆、栈、.data段）。
  - 高端 MCU/DSP 的 **TCM (紧耦合内存)**，用作高速指令/数据缓存。
  - 外部并口 SRAM：高速缓存或 FPGA 配置存储器。

### 3.2 DRAM / SDRAM / DDR SDRAM (动态RAM)
- **DRAM 核心**：一个晶体管 + 一个小电容。依靠电容电荷保存数据，**必须定期刷新**，否则数据丢失。
- **SDRAM (同步DRAM)**：给 DRAM 加上了同步时钟，所有信号在时钟边沿生效，读写效率大幅提升。
- **DDR SDRAM (双倍数据率SDRAM)**：在时钟的上升沿和下降沿各传输一次数据，等效频率翻倍。衍生出 DDR3/4/5、低功耗版 LPDDR4/5、图形版 GDDR。
- **PSRAM (伪静态RAM)**：内部是 DRAM 阵列 + 自刷新控制，对外封装成 SRAM 的并口或 SPI 接口。给不想要复杂 DDR 控制器的 MCU 提供廉价大容量 RAM。

### 3.3 ROM 家族 (只读存储器)
- **Mask ROM**：光罩编程，数据在制造时固化。仅适用于天文数字量级且代码永不更改的产品。
- **PROM**：一次性可编程，熔丝/反熔丝结构。
- **EPROM**：可多次编程，但擦除需紫外线照射整个芯片窗口，已彻底淘汰。
- **EEPROM**：**电擦除，可逐字节读写**。寿命长（约 100 万次擦写），容量小 (KB~几MB)，写入速度慢。**嵌入式设备中存储配置参数、校准值、序列号的“金标准”**，通常走 I²C/SPI 接口。

### 3.4 Flash (闪存)
Flash 本质上是一种块擦除的 EEPROM，因结构不同分为两大流派：

#### NOR Flash
- **访问方式**：带独立地址线或通过高速 SPI，**支持随机读取任意地址**。
- **核心优势**：**支持 XIP (片内执行)**，CPU 可直接在 Flash 上取指令运行，无需复制到 RAM。
- **劣势**：写入/擦除很慢，容量偏小 (几MB~几百MB)，单位成本高。
- **嵌入式应用**：MCU 内部 Flash 基本都是 NOR 型；外部 SPI NOR Flash 常用于存放 bootloader 和固件。

#### NAND Flash
- **访问方式**：存储单元串联，按页 (Page) 读取，按块 (Block) 擦除，不提供随机地址线。
- **特点**：容量极大，成本极低，写入速度比 NOR 快得多。
- **天生缺陷**：存在出厂坏块和使用坏块，**必须依赖 ECC 纠错、坏块管理和磨损均衡算法**。
- **细分与嵌入式应用**：
  - **Raw NAND**：直接由 MPU 的 NAND 控制器驱动，软件 (U-Boot/Linux MTD) 负责管理，常搭配 UBIFS 文件系统。
  - **eMMC**：NAND Flash + 内置控制器，封装成标准 JEDEC 接口，管理层已内置。主流嵌入式系统的“硬盘”。
  - **UFS**：全双工串行，速度远超 eMMC，见于高端移动平台。

> 注：常见的 SPI Flash 绝大多数是 **NOR Flash**，通过 SPI/OSPI 总线通信，兼具小体积与 XIP 能力。

## 4. 为何分类如此复杂？——嵌入式设计的五个核心权衡

没有一种存储器能同时满足 **“快、大、非易失、便宜”**。嵌入式系统对存储的需求是撕裂的，这直接催生了多层次分类：

1. **速度与成本的对决**
   - SRAM 极快，但 1MB 比整颗 MCU 还贵；DRAM 慢且需刷新，但几美金就能买到 GB 级容量。
   - 架构分层：**SRAM 做缓存/TCM，DRAM 做主存**。

2. **易失与持久的天平**
   - RAM 掉电即丢，但可无限次读写；Flash 断电不丢，但有擦写寿命且速度慢。
   - 分工：**Flash 存固件和文件系统，SRAM 跑变量，EEPROM 存配置参数**。

3. **片内与片外的抉择**
   - 简单 MCU 集成 SRAM + NOR Flash 即可单芯片工作。
   - 运行复杂 OS 需外扩 DDR 和 eMMC，必须 SoC 集成专用控制器和高速 PHY，带来功耗与面积成本。

4. **代码执行 vs. 数据存储的截然不同**
   - CPU 取指需随机读取并就地执行 → 只有 **NOR Flash** 能 XIP。
   - 海量数据存写需低成本大容量 → **NAND / eMMC** 是不二之选。
   - 实际板卡上常出现 **NOR (boot) + NAND/eMMC (rootfs)** 的组合。

5. **写粒度与耐久度的取舍**
   - **EEPROM** 可以 **修改单个字节**，寿命百万次，天然适合小量频繁改写的参数。
   - **Flash** 必须 **先擦除整个块 (4KB~128KB)** 才能写入，“改一个字节” 也要搬移整个块，磨损大，不适合细碎修改。因此即便有大容量 Flash，板上常保留一颗小 EEPROM。

## 5. 嵌入式存储选型速查表

| 应用需求 | 推荐方案 | 典型实例 |
|---|---|---|
| 存放并就地执行固件 | 片内 NOR Flash / 外部 SPI NOR Flash | STM32 内置 Flash, W25Qxx |
| 固件运行时 RAM (堆/栈/数据段) | 片内 SRAM / TCM | MCU 内部 SRAM |
| 运行 Linux 等大型系统的内存 | 外扩 DDR3 / DDR4 / LPDDR4 | i.MX6 + DDR3 |
| 大量用户数据、日志、文件系统 | eMMC / Raw NAND / TF 卡 | 摄像头录像存 eMMC |
| 少量配置参数、序列号 | EEPROM / 片内 Flash 模拟 EEPROM | AT24C02 |
| SRAM 容量不足，但不愿上复杂 DDR | 串行/并行 PSRAM | ESP32 挂载 QSPI PSRAM |
| 极速数据缓冲、处理器间共享 | 外部高速同步 SRAM 或 DPRAM | FPGA + DSP 间用双口 RAM |

---

## 6. 小结

存储器的复杂分类，本质是 **物理现实与工程需求博弈后形成的最优层次结构**。从最快的 SRAM 到最持久的 Flash，每一层都精准地弥补了其它层次的不足。作为嵌入式工程师，理解这张 “存储地图”，选型和系统设计时就能游刃有余。
