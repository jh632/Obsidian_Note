# 嵌入式设计模式总览

## 概述

嵌入式系统虽然资源受限，但软件架构的设计模式贯穿始终——从驱动层到应用层，从裸机到 RTOS。
本文档提供一个系统化的学习路线，按**从基础到进阶**的顺序梳理嵌入式中最常见的设计模式。

## 学习路线图

```
第一阶段：无处不在的基础模式（必学）
  状态机模式 ──→ 观察者模式 ──→ 生产者-消费者模式
      │                │                │
      └────────────────┴────────────────┘
                       │
第二阶段：硬件抽象与资源管理（工程核心）
  适配器/外观模式 ──→ 单例模式 ──→ 侵入式链表（已有笔记）
      │                    │              │
      └────────────────────┴──────────────┘
                       │
第三阶段：解耦与扩展（架构进阶）
  桥接模式 ──→ 策略模式 ──→ 工厂模式 ──→ 模板方法模式
      │            │            │              │
      └────────────┴────────────┴──────────────┘
                       │
第四阶段：高级模式（特定场景）
  责任链模式 ──→ 命令模式 ──→ 代理模式 ──→ 中介者模式
```

## 一、状态与行为管理

### [[嵌入式设计模式-状态机|状态机模式]]
- **一句话**：用有限状态驱动系统行为，禁止出现未定义的状态转移
- **嵌入式场景**：OTA 升级流程、按键消抖、通信协议状态（连接/断开/重连）、电源管理
- **你在项目中已用到的**：[[ota升级状态机]] 中的 `IDLE → RECEIVING → FINISHED/ERROR`

### [[策略模式（嵌入式）|策略模式]]
- **一句话**：同一接口，运行时切换不同算法实现
- **嵌入式场景**：多种传感器融合算法可切换、不同加密算法可替换、多种滤波算法（均值/中值/卡尔曼）

### [[命令模式（嵌入式）|命令模式]]
- **一句话**：把请求封装成对象，支持排队、撤销、日志
- **嵌入式场景**：串口命令解析与执行队列、UI 事件队列、传感器采样调度

---

## 二、通信与事件处理

### [[嵌入式设计模式-观察者|观察者模式 / 回调模式]]
- **一句话**：Subject 状态变化时主动通知所有注册的 Observer
- **嵌入式场景**：GPIO 中断回调、传感器数据就绪通知、电池电量变化通知多个模块
- **典型形式**：C 语言中的函数指针回调数组，RTOS 中的事件组/信号量通知

### [[发布订阅模式（嵌入式）|发布-订阅模式]]
- **一句话**：通过消息总线解耦发布者和订阅者（比 Observer 更进一步解耦）
- **嵌入式场景**：CAN/LIN 总线消息分发、BLE 服务特征值广播、MQTT 话题订阅
- **与 Observer 的区别**：Observer 是 Subject 直接通知 Observer；Pub-Sub 通过中间人 EventBus 中转

### [[生产者消费者模式（嵌入式）|生产者-消费者模式]]
- **一句话**：生产者线程填充缓冲区，消费者线程从中取走数据
- **嵌入式场景**：UART 接收 ISR → RingBuffer → 协议解析任务、ADC 采样 ISR → Buffer → 算法处理任务
- **核心组件**：环形缓冲区 + 互斥锁/信号量

### [[责任链模式（嵌入式）|责任链模式]]
- **一句话**：事件沿责任链逐个传递，每个处理器决定处理/传递/丢弃
- **嵌入式场景**：中断级联处理、数据包多级过滤（地址过滤 → CRC校验 → 协议解析）、按键事件分发

---

## 三、硬件抽象与资源管理

### [[适配器模式（嵌入式）|适配器模式 / HAL 层]]
- **一句话**：把不同硬件接口统一定义为一致的 API
- **嵌入式场景**：I2C/SPI/UART 传感器统一成 `sensor_read()` 接口、不同型号 Flash 芯片的读写擦除统一接口
- **这就是 ESP-IDF 的 HAL/BSP 架构核心**

### [[嵌入式设计模式-单例|单例模式]]
- **一句话**：全局唯一的硬件资源只能有一个实例
- **嵌入式场景**：I2C 总线管理器（只有一个实例管理 SDA/SCL）、SPI Flash 分区管理器、日志系统
- **注意**：嵌入式单例通常是"静态全局变量"而非懒加载，因为资源在编译时就确定了

### [[侵入式链表|侵入式链表]]
- 已有笔记 → [[侵入式链表]]
- 嵌入式内存管理的基石，RTOS 线程调度、设备管理、内存池的核心数据结构

### [[对象池模式（嵌入式）|对象池模式]]
- **一句话**：预分配固定数量的对象，用完归还，避免动态分配
- **嵌入式场景**：网络包缓冲区池（避免碎片化）、串口 DMA 描述符池、连接句柄池

## 四、解耦与扩展

### [[嵌入式设计模式-桥接|桥接模式]]
- **一句话**：将抽象（功能）与实现（硬件）分离，两者独立变化
- **嵌入式场景**：传感器驱动框架——上层算法（滤波/融合）不关心底层是 I2C 还是 SPI、显示屏驱动——上层 UI 不关心底层是 SPI LCD 还是 I2C OLED
- **与适配器的区别**：适配器解决接口不匹配问题（已有两个固定接口），桥接在设计时就刻意分离

### [[嵌入式设计模式-工厂|工厂方法模式]]
- **一句话**：把对象的创建延迟到子类/具体实现中
- **嵌入式场景**：根据设备树/配置文件动态创建不同类型的驱动实例、根据检测到的芯片 ID 返回对应的驱动 ops

### [[模板方法模式（嵌入式）|模板方法模式]]
- **一句话**：父类定义算法骨架，子类填充具体步骤
- **嵌入式场景**：设备初始化流程（`power_on → hw_reset → load_config → enable_interrupt`）、
  OTA 流程（`check_version → download → verify → apply → reboot`）

### [[装饰器模式（嵌入式）|装饰器模式]]
- **一句话**：运行时动态给驱动叠加功能
- **嵌入式场景**：给传感器读数叠加 CRC 校验、追加日志记录、追加数据压缩

---

## 五、高级模式

### [[代理模式（嵌入式）|代理模式]]
- **一句话**：用一个代理对象控制对真实对象的访问
- **嵌入式场景**：RPC 桩（本地函数调用伪装成远程设备操作）、延迟初始化的大外设（如 WiFi 模块）

### [[中介者模式（嵌入式）|中介者模式]]
- **一句话**：用一个中介对象来协调多个外设/模块的交互，避免网状依赖
- **嵌入式场景**：电源管理模块协调显示屏、传感器、WiFi 的休眠/唤醒、多传感器融合 hub
