# OpenCode 常用指令

## 概述
OpenCode 代理框架通过一组结构化工具（Tools）与环境交互。掌握这些指令是高效使用代理完成编码、调试和研究任务的基础。

## 核心概念
- **工具即接口**：代理不直接操作文件系统，而是通过声明式工具调用完成读写、执行、搜索等操作。
- **隐式顺序控制**：单条消息中多个无依赖的工具调用会自动并行执行，提升效率。
- **读取前置原则**：修改任何文件前必须先 `read`，确保基于最新内容做精准替换。

## 工具分类与细节

### 1. 文件读写
| 工具 | 作用 | 关键约束 |
|------|------|----------|
| `read` | 读取文件内容或列出目录结构，支持指定起始行和最大行数 | 长文件建议分段读取 |
| `write` | 创建新文件或覆盖已有文件 | **必须先 `read` 已存在的文件** |
| `edit` | 对已有文件做精确字符串替换 | 只能替换唯一匹配，否则报错；不支持正则 |
| `glob` | 按 glob 模式搜索文件路径（如 `**/*.c`） | 按修改时间排序返回 |
| `grep` | 按正则表达式搜索文件内容 | 返回文件路径和行号，不返回完整内容 |

**使用要点：**
- 修改现有代码时，优先用 `edit` 做最小化替换，避免 `write` 全量覆盖导致无关变更。
- `glob` 和 `grep` 适合快速定位目标文件，之后用 `read` 查看细节。
- 文件名含空格时必须用双引号包裹路径参数。

### 2. 执行与任务编排
| 工具 | 作用 | 典型场景 |
|------|------|----------|
| `bash` | 执行 PowerShell / Shell 命令 | 编译、运行测试、git 操作、npm install |
| `task` | 启动子代理处理复杂多步骤任务 | 深入代码库分析、跨文件重构、批量修改 |
| `delegate` | 后台异步委托任务，不阻塞当前会话 | 长时间运行的研究任务、并行探索 |

**使用要点：**
- `bash` 默认在当前工作目录执行，可通过 `workdir` 参数切换目录，避免在命令中写 `cd`。
- 多步骤依赖任务用 `&&` 链式执行；独立任务拆分为多个并行 `bash` 调用。
- `task` 适合需要上下文记忆的复杂分析；`delegate` 适合不需要即时结果的异步工作。

### 3. 网络与信息获取
| 工具 | 作用 | 关键约束 |
|------|------|----------|
| `webfetch` | 获取指定 URL 的网页内容 | 支持 `markdown`/`text`/`html` 格式；HTTP 自动升级为 HTTPS |

**使用要点：**
- 优先用 `webfetch` 而非 `bash` + `curl` 获取网页，输出更结构化。
- 如果存在更专业的工具（如项目自带的文档查询工具），优先使用专用工具。

### 4. 项目记忆与规划
| 工具 | 作用 | 典型场景 |
|------|------|----------|
| `memory` | 添加/搜索/管理项目级知识库 | 保存技术决策、架构约定、调试经验，跨会话持久化 |
| `todowrite` | 创建并维护当前会话的任务清单 | 多步骤任务跟踪，明确进行中/已完成状态 |

**使用要点：**
- `memory` 的搜索范围可以是 `project`（当前项目）或 `all-projects`（全局）。
- `todowrite` 适合 3 步以上的非平凡任务，避免为简单操作增加管理负担。
- 任务状态流转：`pending` → `in_progress` → `completed`。

### 5. 交互与确认
| 工具 | 作用 | 典型场景 |
|------|------|----------|
| `question` | 向用户提出选择题或开放性问题 | 需求不明确时确认实现方案、选择技术路径 |

**使用要点：**
- 存在多种理解方式时，用 `question` 列出所有选项，不要默认选择。
- 实现前发现需求不清晰，优先提问而非事后修补。

## 使用模式总结

### 高效并行模式
```
# 独立搜索并行执行
glob("**/*.c") + grep("function_name") + webfetch(url)

# 独立读取并行执行
read(file_a) + read(file_b) + read(file_c)
```

### 顺序依赖模式
```
# 先读再改
read(file) → edit(file) → bash(test)

# bash 链式依赖
bash("git add . && git commit -m 'msg' && git push")
```

### 探索-修改-验证循环
```
1. grep/glob 定位目标
2. read 理解上下文
3. edit 做最小修改
4. bash 运行测试验证
5. 如有问题，回到步骤 2
```
